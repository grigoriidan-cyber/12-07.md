## Задание 1

### Active master + passive slave

В этой схеме один сервер основной (master), а второй резервный (slave).
Все запросы идут на master, а slave просто копирует данные с него.

Плюсы:

1. Если master перестанет работать, можно переключиться на slave;
2. Данные дублируются, а это надёжность;
3. Ну и можно отнести обслуживание сервера.

### Master + несколько slave

Здесь master отвечает за запись данных, а slave-серверы можно использовать для чтения.

Плюсы:

1. Уменьшаем нагрузка на master;
2. Система может обслуживать больше пользователей;
3. Если один slave отлетит, остальные продолжат обслуживать запросы, а нагрузка распределится между ними.
